Циклы while и for

При написании скриптов зачастую встает задача сделать однотипное действие много раз.

Для многократного повторения одного участка кода предусмотрены циклы.

Цикл «while»

Цикл while имеет следующий синтаксис:

while (condition) {
  // код
  // также называемый 'телом цикла'
}

Код из тела цикла выполняется, пока условие condition истинно.

Любое выражение или переменная может быть условием цикла: условие while вычисляется и преобразуется в логическое значение.

Цикл «do...while»

Проверку условия можно разместить под телом цикла, используя специальный синтаксис do...while:

do {
  // тело цикла
} while (condition);

Цикл сначала выполнит тело, а затем проверит условие condition, и пока его значение равно true, он будет выполняться снова и снова.

Такая форма синтаксиса оправдана, если вы хотите, чтобы тело цикла выполнилось хотя бы один раз, даже если условие окажется ложным.

Цикл «for»

Более сложный, но при этом самый распространенный цикл - цикл for.

Выглядит он так:

for (начало; условие; шаг) {
  // тело цикла
}

Рассмотрим конструкцию for подробней:

Начало let i = 0 Выполняется один раз при входе в цикл.
Условие i < 3 Проверяется перед каждой итерацией цикла. Если оно вычислится в false, цикл остановится.
Тело alert(i) Выполняется снова и снова, пока условие вычисляется в true.
Шаг i++ Выполняется после тела цикла на каждой итерации перед проверкой условия.

То есть, начало выполняется один раз, а затем каждая итерация заключается в проверке условия, после которой выполняется тело и шаг.

Встроенное объявление переменной

Обычно переменная счетчика i объявляется прямо в цикле. Это так называемое «встроенное» объявление переменной. Такие переменные существуют только внутри цикла.

Вместо объявления новой переменной мы можем использовать уже существующую. В таком случае переменная будет доступна и вне цикла тоже.

Пропуск частей «for»

Любая часть for может быть пропущена.

Прерывание цикла: «break»

Обычно цикл завершается при вычислении условия в false.

Но мы можем выйти из цикла в любой момент с помощью специальной директивы break.

Переход к следующей итерации: continue

Директива continue - «облегченная версия» break. При ее выполнении цикл не прерывается, а переходит к следующей итерации (если условие все еще равно true).

Нельзя использовать break/continue справа от оператора «?»

Обратите внимание, что эти синтаксические конструкции не являются выражениями и не могут быть использованы с тернарным оператором ?.

Метки для break/continue

Бывает, нужно выйти одновременно из нескольких уровней цикла сразу.

Достичь желаемого поведения можно с помощью меток.

Метка имеет вид идентификатора с двоеточием перед циклом:

label_name: for (...) {
   ...
}

Вызов break label_name в цикле ищет ближайший цикл с такой меткой и переходит в его конец.

Можно размещать метку на отдельной строке:

label:
for (let i = 0; i < 10; i++) {...}

Директива continue также может быть использована с меткой. В таком случае управление перейдет на следующую итерацию цикла с меткой.

Метки не позволяют «прыгнуть» куда угодно

Метки не дают возможность передавать управление в произвольное место кода.

Директива break должна находиться внутри блока кода. Технически, подойдет любой маркированный блок кода, например:

label: {
   // ...
   break label; // работает
   // ...
}

... Хотя в 99% случаев break используется внутри циклов.

К слову, continue возможно только внутри цикла.

Итого

Мы рассмотрели 3 вида циклов:

  • while - Проверяет условие перед каждой итерацией;
  • do...while - Проверят условие после каждой итерации;
  • for (;;) - Проверяет условие перед каждой итерацией, есть возможность задать дополнительные настройки.

Чтобы организовать бесконечный цикл, используют конструкцию while (true). При этом он, как и любой другой цикл, может быть прерван директивой break.

Если на данной итерации цикла делать больше ничего не надо, но полностью прекращать цикл не следует - используют директиву continue.

Обе этих директивы поддерживают метки, которые ставятся перед циклом. Метки - единственный способ для break/continue выйти за пределы текущего цикла, повлиять на выполнение внешнего.

Заметим, что метки не позволяют прыгнуть в произвольное место кода, в JavaScript нет такой возможности.
